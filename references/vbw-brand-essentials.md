# VBW Brand Vocabulary

## Semantic Symbols

| Meaning          | Symbol | Unicode  | Usage                    |
|------------------|--------|----------|--------------------------|
| Success/complete | ✓      | U+2713   | Task done, check passed  |
| Failure/error    | ✗      | U+2717   | Task failed, check error |
| In progress      | ◆      | U+25C6   | Currently executing      |
| Pending/queued   | ○      | U+25CB   | Waiting to start         |
| Action/lightning | ⚡     | U+26A1   | Command invocation       |
| Warning          | ⚠      | U+26A0   | Non-blocking concern     |
| Info/arrow       | ➜      | U+279C   | Navigation, next step    |

## Horizontal Bars (LLM-generated output)

Character: ━ (U+2501, BOX DRAWINGS HEAVY HORIZONTAL)

Use horizontal bars as header/footer delimiters when the content is generated by the LLM (commands, agents). Plain markdown body between bars — no side borders, no padding math. LLMs cannot reliably align box-drawing side borders, so bars avoid the misalignment problem entirely.

Example:
```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  CHECKPOINT 1/3 — 01-01: Core Feature
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Scenario description here.

**Expected:** Expected result here.
```

## Box Drawing (script-generated output only)

Use box-drawing characters only in script-generated output where alignment is deterministic (bash `printf` with fixed widths). Never use side borders (`│`, `║`) in LLM-generated content.

### Critical / Phase-level (double-line)

Characters: ╔ (U+2554) ═ (U+2550) ╗ (U+2557) ║ (U+2551) ╚ (U+255A) ╝ (U+255D)

### Standard / Task-level (single-line)

Characters: ┌ (U+250C) ─ (U+2500) ┐ (U+2510) │ (U+2502) └ (U+2514) ┘ (U+2518)

## Progress Bars

Format: `[filled][empty]` using block elements.

- Filled: █ (U+2588)
- Empty: ░ (U+2591)

Pair with percentage: `███████░░░ 70%`

Progress bars always 10 characters wide for visual consistency.

## Rules

1. No ANSI color codes -- not rendered in Claude Code model output
2. No Nerd Font glyphs -- not universally available
3. LLM-generated output uses horizontal bars (━), not box-drawing side borders
4. Keep lines under 80 characters inside boxes
5. Use semantic symbols consistently across all agent output
6. When using AskUserQuestion, always output 3–4 blank lines after your text and before the tool call — the AskUserQuestion dialog obscures the last few visible lines (ref: anthropics/claude-code#23862)

This file is the single source of truth for brand vocabulary. Output templates are defined inline in each command file's Output Format section.
